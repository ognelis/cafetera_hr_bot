## 1. Что нужно знать перед запуском

### Из чего состоит система

Система состоит из трёх основных частей:

- **Админ-панель** — сайт, где вы загружаете документы (инструкции, регламенты), которые будет знать бот. Открывается в браузере по адресу `http://localhost:8000/documents`. Админ-панель — лёгкий сервис: загружает файлы в S3 и передаёт задачу на обработку RAG-сервису.
- **VK-бот** — бот в вашем сообществе ВКонтакте, который отвечает на вопросы сотрудников по загруженным документам.
- **RAG-сервис** — внутренний микросервис, который обрабатывает полный конвейер документов: скачивает файл из S3, парсит (Docling), разбивает на части, создаёт эмбеддинги и индексирует в Qdrant. Также обрабатывает запросы к ИИ и базе знаний. Админ-панель и VK-бот общаются с ним по HTTP. Работает на порту `8001`. Запускается автоматически скриптами.

### Что такое ИИ-провайдер и зачем он нужен

Для того чтобы бот «понимал» вопросы и давал умные ответы, нужны две ИИ-модели:

- **LLM (языковая модель)** — «мозг» бота, который читает вопрос и формулирует ответ. Похоже на ChatGPT, но может работать локально.
- **Embedding-модель** — модель, которая «понимает» смысл текста и помогает найти нужный документ среди загруженных. Работает «в фоне», пользователь её не видит.

Доступны три варианта, где запускать эти модели:

| Провайдер | Как работает | Стоимость | Требования |
|---|---|---|---|
| **llama.cpp** (рекомендуется) | Модель работает локально, максимальный контроль над памятью | Бесплатно | Нужна ручная установка llama.cpp, файлы моделей в формате `.gguf` |
| **Ollama** | Модель работает прямо на вашем компьютере | Бесплатно | ОЗУ от 8 ГБ, при первом запуске скачивает ~4–6 ГБ |
| **OpenAI** | Модель работает в облаке OpenAI (как ChatGPT) | Платно (по API-ключу) | Нужен интернет и ключ API от openai.com |

**Если вы запускаете впервые — выбирайте llama.cpp** (вариант по умолчанию, лучший контроль памяти).

## 2. Установить Docker

Docker — программа, которая запускает вспомогательные сервисы (базу данных и файловое хранилище) в изолированных контейнерах. Вам не нужно разбираться в том, как это работает — Docker всё сделает сам.

### macOS

1. Откройте браузер, перейдите на https://www.docker.com/products/docker-desktop/
2. Нажмите **«Download for Mac»**.
   - Если у вас Mac на M1/M2/M3/M4 — выберите **Apple Silicon**.
   - Если у вас старый Mac на Intel — выберите **Intel**.
3. Откройте скачанный `.dmg` файл.
4. Перетащите иконку Docker в папку **Программы** (Applications).
5. Запустите **Docker** из папки Программы. При первом запуске macOS может попросить подтвердить открытие — нажмите **«Открыть»**.
6. Дождитесь, пока иконка Docker в верхней панели (рядом с часами) перестанет мигать — это значит Docker готов к работе.

### Linux (Debian/Ubuntu)

Откройте **Терминал** и выполните команды по одной:

```bash
# Обновить список пакетов
sudo apt-get update

# Установить зависимости
sudo apt-get install -y ca-certificates curl gnupg

# Добавить ключ Docker
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# Добавить репозиторий Docker
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Установить Docker
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Разрешить запуск Docker без sudo
sudo usermod -aG docker $USER
```

**После последней команды выйдите из системы и войдите обратно** (или перезагрузите компьютер).

Проверьте, что Docker работает:

```bash
docker --version
```

Должна появиться строка вида `Docker version 27.x.x ...`

## 3. Установить uv

`uv` — менеджер Python-пакетов. Он автоматически скачает нужную версию Python и все зависимости проекта. Нужен только для вариантов запуска через bash-скрипты (варианты А и В).

### macOS и Linux

Откройте **Терминал** и выполните:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

После установки **закройте и откройте Терминал заново**, чтобы команда `uv` стала доступна.

Проверьте установку:

```bash
uv --version
```

Должна появиться строка вида `uv 0.x.x`.

> Для вариантов Б и Г (запуск через Docker) `uv` не нужен.

### 4. Установить ИИ-провайдер

Нужно выбрать **один** из трёх вариантов запуска ИИ-моделей. Если сомневаетесь — выбирайте **llama.cpp** (раздел 4.3), это самый контролируемый путь с лучшим управлением памятью.

| Провайдер | Когда выбирать | Что нужно установить |
|---|---|---|
| **llama.cpp** | Рекомендуется, максимальный контроль памяти | `llama-server` + файлы моделей |
| **Ollama** | Простой запуск, без ручной настройки | Приложение Ollama |
| **OpenAI** | Нет мощного железа, готовы платить | Только API-ключ |

Подробное сравнение провайдеров — см. **[docs/providers.md](docs/providers.md)**.

> **Совет:** у каждого провайдера свой набор применимых `.env`-переменных. Часть настроек (например, `LLM_PRESENCE_PENALTY` для Ollama или `EMBED_POOLING` для OpenAI) **молча игнорируется**. Матрицу совместимости см. в [docs/providers.md, раздел «Какие `.env`-переменные применимы к каждому провайдеру»](docs/providers.md#какие-env-переменные-применимы-к-каждому-провайдеру).

### 4.1 Ollama (рекомендуется)

Ollama — программа, которая скачивает и запускает ИИ-модели прямо на вашем компьютере.

#### macOS

1. Перейдите на https://ollama.com/download
2. Нажмите **«Download for macOS»**.
3. Откройте скачанный файл и установите Ollama.
4. Запустите **Ollama** из папки Программы.

#### Linux (Debian/Ubuntu)

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

Проверьте установку:

```bash
ollama --version
```

> **Важно:** При первом запуске скрипта Ollama автоматически скачает языковую модель (~2–3 ГБ) и embedding-модель (~2–3 ГБ). Это занимает время — зависит от скорости интернета. Дождитесь завершения.

При запуске любого `run_*.sh`-скрипта просто нажмите **Enter** в ответ на вопрос о провайдере — по умолчанию выбран llama.cpp. Скрипт сам скачает модели и запустит серверы.

---

### 4.2 OpenAI

Модели работают в облаке OpenAI. Ничего устанавливать на свой компьютер не нужно — требуется только API-ключ и интернет.

#### Получить API-ключ

1. Зарегистрируйтесь на https://platform.openai.com/
2. Пополните баланс (оплата по факту использования токенов).
3. Откройте https://platform.openai.com/api-keys и создайте ключ вида `sk-...`.
4. Скопируйте ключ — он показывается только один раз.

#### Как использовать

Есть два способа:

**Способ 1 — интерактивно при запуске (рекомендуется).** Запустите любой `run_*.sh`-скрипт и в ответ на вопрос о провайдере введите `2`. Скрипт попросит вставить API-ключ.

**Способ 2 — зафиксировать в `.env`.** Добавьте в файл `.env`:

```
LLM_PROVIDER=openai
LLM_MODEL=gpt-4o-mini
LLM_API_KEY=sk-...ваш-ключ...
EMBEDDING_PROVIDER=openai
EMBEDDING_MODEL=text-embedding-3-small
EMBEDDING_API_KEY=sk-...ваш-ключ...
```

> **Важно:** не публикуйте `.env` с реальным ключом — он даёт доступ к оплаченному балансу. Файл `.env` уже добавлен в `.gitignore`.

Можно смешивать провайдеров: например, LLM через OpenAI (качественнее), а Embedding — через Ollama (бесплатно). Подробнее — [docs/providers.md](docs/providers.md).

### 4.3 llama.cpp (рекомендуется)

llama.cpp даёт максимальный контроль над локальными моделями в формате `.gguf`. Подходит, если нужно ограничить потребление памяти или использовать свою модель.

**Преимущества llama.cpp:**
- Квантизация KV Cache (`q8_0`) — экономия 50% VRAM
- Контроль параллельных запросов (`LLM_PARALLEL`)
- Гибкая настройка контекста и batch sizes
- Полная конфиденциальность (всё локально)

#### Шаг 1. Установить `llama-server`

**macOS:**

```bash
brew install llama.cpp
```

> Если Homebrew не установлен — см. <https://brew.sh>.

**Linux (Debian/Ubuntu):**

```bash
sudo apt-get install -y build-essential cmake git
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp
cmake -B build
cmake --build build --config Release -j$(nproc)
sudo cp build/bin/llama-server /usr/local/bin/
```

Проверьте установку:

```bash
llama-server --version
```

#### Шаг 2. Файлы моделей

При первом запуске скрипты **автоматически скачают нужные `.gguf`-файлы** в папку `models/`. Ничего делать вручную не нужно.

Если хотите подготовить модели заранее — см. [docs/llamacpp.md, раздел 2](docs/llamacpp.md#2-файлы-моделей-gguf).

#### Шаг 3. Запуск

При запуске любого `run_*.sh`-скрипта в ответ на вопрос о провайдере введите `3` (llamacpp). Скрипт сам:

- Скачает файлы моделей (при первом запуске).
- Запустит `llama-server` для LLM на порту **8080**.
- Запустит `llama-server` для Embedding на порту **8090**.
- При необходимости запустит реранкер на порту **8082**.
- Определит тип GPU (Metal на Mac, CUDA на Linux) и включит ускорение.

#### Ручной запуск серверов (опционально)

Если нужно поднять серверы отдельно (например, в отдельном окне терминала):

```bash
bash scripts/run_llama_llm.sh           # LLM, порт 8080
bash scripts/run_llama_embeddings.sh    # Embedding, порт 8090
bash scripts/run_llama_reranker.sh      # Reranker, порт 8082 (опционально)
```

Полная документация по llama.cpp (GPU-ускорение, другие модели, реранкер, настройки pooling) — см. **[docs/llamacpp.md](docs/llamacpp.md)**. Гайд по параметрам для не-ML специалистов — **[docs/llama-params-guide.md](docs/llama-params-guide.md)**.

## 6. Настроить файл .env

Файл `.env` — это файл с настройками. Скрипты и Docker читают настройки именно из него.

```bash
cd ~/Projects/cafetera_hr_bot    # перейти в папку проекта
cp .env.example .env             # создать файл настроек из шаблона
```

Теперь откройте `.env` для редактирования:

### macOS

```bash
open -e .env
```

### Linux

```bash
nano .env
```

> В nano: редактируйте текст как обычно. Для сохранения нажмите `Ctrl+O`, затем `Enter`. Для выхода — `Ctrl+X`.

### Что нужно обязательно заполнить

| Параметр | Что вписать | Для чего |
|---|---|---|
| `ADMIN_API_KEY` | Любой длинный пароль, например `my-secret-key-2025` | Вход в админ-панель |
| `VK_ACCESS_TOKEN` | Токен вашего VK-сообщества | Нужен для VK-бота |
| `VK_GROUP_ID` | ID вашего VK-сообщества (число) | Нужен для VK-бота |

> **VK-токен и ID группы** нужны только если вы запускаете VK-бота. Для работы только с админ-панелью их можно не заполнять.

> **RAG-сервис:** Настройки `RAG_SERVICE_URL` и `RAG_SERVICE_API_KEY` в `.env` обычно менять не нужно — скрипты подставляют значения автоматически. Если RAG-сервис запускается на другом адресе, укажите его в `RAG_SERVICE_URL`.

### Как получить VK_ACCESS_TOKEN и VK_GROUP_ID

1. Зайдите в своё сообщество ВКонтакте.
2. Перейдите в **Управление → Настройки → Работа с API**.
3. Нажмите **«Создать ключ»** и скопируйте токен.
4. `VK_GROUP_ID` — это числовой ID вашей группы (можно увидеть в адресной строке или в настройках группы).

### Настройка ИИ-провайдера (необязательно)

При каждом запуске скрипт **всегда** показывает интерактивное меню выбора провайдера (Ollama / OpenAI / llama.cpp / custom URL). Это поведение по умолчанию.

Значения `LLM_PROVIDER` и `EMBEDDING_PROVIDER` в `.env` **не отключают** меню — они лишь задают вариант, который подставится по умолчанию. Чтобы согласиться с ним, достаточно нажать **Enter**. Если в `.env` выбрана `openai`, то при нажатии Enter скрипт возьмёт ключ `LLM_API_KEY` / `EMBEDDING_API_KEY` из `.env` и не будет спрашивать его интерактивно.

Подробнее о переменных на провайдера — см. **[docs/providers.md](docs/providers.md)**.

#### Полностью отключить меню: `SKIP_PROVIDER_MENU`

Для CI, systemd-юнитов и любого запуска без терминала добавьте в `.env`:

```
SKIP_PROVIDER_MENU=true
```

Тогда скрипты `run_admin.sh`, `run_admin_docker.sh`, `run_all.sh`, `ragas/run.sh` **не покажут меню вообще** и возьмут все значения из `.env`. Убедитесь, что критичные поля заданы заранее:

| Провайдер | Обязательные поля в `.env` | Желательные |
|---|---|---|
| **Ollama** | `LLM_PROVIDER=ollama`, `EMBEDDING_PROVIDER=ollama` | `LLM_MODEL`, `EMBEDDING_MODEL` |
| **OpenAI** | `LLM_PROVIDER=openai`, `LLM_API_KEY`, `EMBEDDING_PROVIDER=openai`, `EMBEDDING_API_KEY` | `LLM_MODEL`, `EMBEDDING_MODEL` |
| **llama.cpp** | `LLM_PROVIDER=llamacpp`, `EMBEDDING_PROVIDER=llamacpp` | `EMBED_POOLING` (для не Qwen3-моделей) |

Если для `openai` не заданы API-ключи — скрипт завершится с явной ошибкой, не стартуя сервисы. Полные примеры `.env` для каждого провайдера — см. [docs/providers.md, раздел «Полностью неинтерактивный запуск»](docs/providers.md#полностью-неинтерактивный-запуск-skip_provider_menu).

Для варианта **llama.cpp** дополнительно нужен `llama-server` и `.gguf`-файлы моделей — см. **[docs/llamacpp.md](docs/llamacpp.md)**.

> **Совет:** Если вы не разбираетесь в ML — прочитайте **[docs/llama-params-guide.md](docs/llama-params-guide.md)**. Там простым языком объяснено, как подобрать параметры под ваше железо и количество пользователей.

> **Важно:** URL-адреса провайдеров (`LLM_BASE_URL`, `EMBEDDING_BASE_URL`, `RERANKER_URL`) настраиваются автоматически — задавать их в `.env` не нужно. Скрипты сами определяют правильные адреса в зависимости от выбранного провайдера и режима запуска (локальный или Docker).

Остальные параметры (`DATABASE_URL`, `QDRANT_URL`, `S3_ENDPOINT_URL`) оставьте как есть — скрипты и Docker подставят правильные значения автоматически.

### Вариант А — только Админ-панель (bash-скрипт)

Самый простой способ для первого запуска на своём компьютере.

**Требования:** Docker, uv, Ollama (или OpenAI API-ключ, или llama.cpp + файлы моделей).

```bash
cd ~/Projects/cafetera_hr_bot
bash scripts/run_admin.sh
```

**Что произойдёт:**

1. Скрипт проверит, что Docker и uv установлены.
2. **Спросит, какой LLM-провайдер использовать** — нажмите **Enter** для выбора llama.cpp.
3. **Спросит, какой Embedding-провайдер использовать** — снова нажмите **Enter**.
4. Установит Python-зависимости (1–2 минуты при первом запуске).

**Требования:** Docker, Ollama запущена на хосте (или OpenAI API-ключ прописан в `.env`).

**Запуск всего стека:**

```bash
cd ~/Projects/cafetera_hr_bot
docker compose up -d --build
```

Это запустит:
- **PostgreSQL** — база данных
- **Qdrant** — база знаний для поиска по документам
- **MinIO** — файловое хранилище
- **RAG-сервис** — микросервис для ИИ и поиска (порт 8001)
- **Админ-панель** — веб-интерфейс (порт 8000)
- **VK-бот** — бот в режиме polling

Посмотреть что происходит (логи):

```bash

docker compose logs -f admin        # только админка
docker compose logs -f vk_bot       # только VK-бот
docker compose logs -f rag-service   # только RAG-сервис
```

Остановить все сервисы:

```bash

```

**Запустить только нужные части:**

```bash

```

> **Примечание об Ollama в Docker:** Ollama должна быть запущена на вашем компьютере (хосте). Docker-контейнеры обращаются к ней по адресу `host.docker.internal:11434` автоматически — ничего дополнительно настраивать не нужно.

## 10. Оценка качества ответов (RAGAS)

В проекте есть автоматическая оценка качества на базе [RAGAS](https://docs.ragas.io/en/stable/concepts/metrics/available_metrics/). Все режимы запускаются одним скриптом `bash ragas/run.sh` — скрипт покажет интерактивное меню:

| № | Режим | Что делает |
|---|---|---|
| 1 | `generate` | Генерирует тестовые вопросы из загруженных документов. |
| 2 | `evaluate` | Прогоняет готовый тестсет через RAG и считает метрики RAGAS. |
| 3 | `all` *(по умолчанию)* | Генерация + оценка одной командой. |
| 4 | `retrieval` | Независимая оценка качества **поиска** на публичном бенчмарке [SberQuAD](https://huggingface.co/datasets/kuznetsoffandrey/sberquad) — LLM не требуется. |
| 5 | `optimize` | **Автоматический подбор системного промпта:** LLM пишет варианты, RAGAS их оценивает, лучший сохраняется в `ragas/best_prompt.txt`. |

### Примеры

```bash
# Стандартный цикл: генерация тестсета + оценка RAG-конвейера
bash ragas/run.sh

# Быстрая оценка поиска (без LLM) на 200 вопросах в режиме dense
bash ragas/run.sh retrieval --k 10 --size 200 --mode dense

# Подбор лучшего системного промпта: 15 вариантов на выборке из 10 вопросов
bash ragas/run.sh optimize --variants 15 --sample-size 10
```

### Считаемые метрики

- **Faithfulness** — не выдумал ли бот факты, которых нет в документах.
- **Context Precision** — доля полезных кусков среди найденных.
- **Answer Relevancy** — ответил ли бот на тот вопрос.
- **NonLLM Context Recall** *(если в тестсете есть `reference_contexts`)* — нашёл ли поиск все нужные эталонные куски. Без вызовов LLM — стабильно на слабых локальных моделях.
- **Semantic Similarity** *(если в тестсете есть `reference`)* — смысловая близость ответа бота к эталону (только эмбеддинги, без LLM).
- **Retrieval-режим** добавляет: MRR@k, NDCG@k, Hit Rate@k, Recall@k, Precision@k.

Результаты: `ragas/scores.json` (стандартная оценка), `ragas/retrieval_scores.csv` (retrieval), `ragas/best_prompt.txt` + `ragas/optimization_results.json` (optimize).

Подробное описание всех режимов и метрик простым языком — см. **[docs/ragas.md](docs/ragas.md)**.
