# Cafetera HR Bot — Запуск админ-панели

Пошаговая инструкция для запуска админ-панели на **macOS** и **Linux (Debian/Ubuntu)**.

---

## 1. Установить Docker

Docker нужен для запуска вспомогательных сервисов (база данных Qdrant и файловое хранилище MinIO).

### macOS

1. Откройте браузер и перейдите на https://www.docker.com/products/docker-desktop/
2. Нажмите **"Download for Mac"** (выберите вариант **Apple Silicon** если у вас Mac на M1/M2/M3/M4, или **Intel** для старых маков).
3. Откройте скачанный `.dmg` файл.
4. Перетащите иконку Docker в папку **Applications** (Программы).
5. Запустите **Docker** из папки Applications. При первом запуске macOS может попросить подтвердить открытие — нажмите **"Открыть"**.
6. Дождитесь, пока иконка Docker в верхней панели (рядом с часами) перестанет мигать — это значит Docker готов к работе.

### Linux (Debian/Ubuntu)

Откройте **Терминал** и выполните команды по одной (копируйте и вставляйте):

```bash
# Обновить список пакетов
sudo apt-get update

# Установить зависимости
sudo apt-get install -y ca-certificates curl gnupg

# Добавить ключ Docker
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# Добавить репозиторий Docker
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Установить Docker
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Разрешить запуск Docker без sudo
sudo usermod -aG docker $USER
```

**После последней команды нужно выйти из системы и войти обратно** (или перезагрузить компьютер), чтобы изменения вступили в силу.

Проверьте, что Docker работает:

```bash
docker --version
```

Должна появиться строка вида `Docker version 27.x.x ...`

---

## 2. Установить uv (менеджер Python-пакетов)

`uv` автоматически скачает нужную версию Python и все зависимости проекта.

### macOS и Linux

Откройте **Терминал** и выполните:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

После установки **закройте и откройте Терминал заново**, чтобы команда `uv` стала доступна.

Проверьте установку:

```bash
uv --version
```

Должна появиться строка вида `uv 0.x.x`.

---

## 3. Установить Ollama (локальная AI-модель)

Ollama нужен для запуска языковой модели и модели эмбеддингов прямо на вашем компьютере (без интернета, бесплатно). Если вы планируете использовать OpenAI вместо локальных моделей — этот шаг можно пропустить.

### macOS

1. Перейдите на https://ollama.com/download
2. Нажмите **"Download for macOS"**.
3. Откройте скачанный файл и установите Ollama.
4. Запустите **Ollama** из папки Applications.

### Linux (Debian/Ubuntu)

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

Проверьте установку:

```bash
ollama --version
```

## 4. Скачать проект

Если у вас уже есть папка с проектом — пропустите этот шаг.

Откройте **Терминал** и выполните:

```bash
cd ~/Projects   # или любая другая папка, куда хотите положить проект
git clone <URL-репозитория> cafetera_hr_bot
cd cafetera_hr_bot
```

> Если `git` не установлен:
> - **macOS**: при первом вызове `git` система предложит установить инструменты разработчика — согласитесь.
> - **Linux**: выполните `sudo apt-get install -y git`

---

## 5. Настроить переменные окружения (.env)

Скрипт `run_admin.sh` читает настройки из файла `.env` в корне проекта.

```bash
cd ~/Projects/cafetera_hr_bot    # перейти в папку проекта
cp .env.example .env             # создать файл настроек из шаблона
```

Теперь отредактируйте `.env`:

### macOS

```bash
open -e .env    # откроет в TextEdit
```

### Linux

```bash
nano .env       # откроет в текстовом редакторе nano
```

> В nano: редактируйте текст как обычно. Для сохранения нажмите `Ctrl+O`, затем `Enter`. Для выхода — `Ctrl+X`.

**Что нужно изменить в файле:**

| Параметр | Что вписать | Обязательно? |
|---|---|---|
| `ADMIN_API_KEY` | Любой длинный случайный пароль, например `my-super-secret-key-12345` | Да |
| `VK_ACCESS_TOKEN` | Токен вашего VK-сообщества (если используете VK-бота) | Нет |

Остальные параметры можно оставить как есть — значения по умолчанию подходят для локальной работы.

---

## 6. Запустить админ-панель

Убедитесь, что **Docker запущен** (на macOS — иконка Docker в верхней панели активна).

Откройте **Терминал** и выполните:

```bash
cd ~/Projects/cafetera_hr_bot
bash scripts/run_admin.sh
```

### Что произойдёт при запуске

Скрипт выполнит всё автоматически:

1. Проверит, что Docker и uv установлены.
2. **Спросит, какой LLM-провайдер использовать** — просто нажмите `Enter` для выбора Ollama (по умолчанию).
3. **Спросит, какой Embedding-провайдер использовать** — снова нажмите `Enter`.
4. Установит Python-зависимости (может занять 1–2 минуты при первом запуске).
5. Запустит Docker-контейнеры (Qdrant и MinIO).
6. Запустит Ollama и скачает нужные модели (при первом запуске скачивание ~2–4 ГБ, займёт время в зависимости от скорости интернета).
7. Запустит веб-сервер админ-панели.

### Когда увидите строки вида:

```
[admin] ==========================================
[admin] Starting admin server
[admin] ==========================================
[admin] Admin UI:    http://127.0.0.1:8000/documents
```

— сервер готов!

---

## 7. Открыть админ-панель

Откройте браузер и перейдите по адресу:

**http://127.0.0.1:8000/documents**

Логин: введите значение `ADMIN_API_KEY` из файла `.env`.

---

## 8. Остановить сервер

Для остановки нажмите **`Ctrl+C`** в Терминале, где запущен скрипт. Он автоматически остановит все фоновые процессы и Docker-контейнеры.

## Частые проблемы

### «docker: command not found»
Docker не установлен или не запущен. Вернитесь к шагу 1.

### «uv: command not found»
Закройте Терминал и откройте заново. Если не помогло — повторите шаг 2.

### «ADMIN_API_KEY is not set»
Откройте файл `.env` и убедитесь, что строка `ADMIN_API_KEY=` содержит значение (не пустая).

### «Qdrant failed to start» или «MinIO failed to start»
- Убедитесь, что Docker запущен.
- Проверьте, не заняты ли порты 6333 или 9000 другими программами:
  ```bash
  lsof -i :6333
  lsof -i :9000
  ```
- Посмотрите логи Docker: `docker compose logs`

### «Ollama is not installed»
Вернитесь к шагу 3 и установите Ollama.

### Модель скачивается слишком долго
При первом запуске Ollama скачивает модели (~2–4 ГБ). Это нормально. Дождитесь завершения.

### Порт 8000 уже занят
Запустите на другом порту:
```bash
ADMIN_PORT=8080 bash scripts/run_admin.sh
```
Тогда админ-панель будет доступна по адресу http://127.0.0.1:8080/documents
