## 1. Что нужно знать перед запуском

### Из чего состоит система

Система состоит из двух основных частей:

- **Админ-панель** — сайт, где вы загружаете документы (инструкции, регламенты), которые будет знать бот. Открывается в браузере по адресу `http://localhost:8000/documents`.
- **VK-бот** — бот в вашем сообществе ВКонтакте, который отвечает на вопросы сотрудников по загруженным документам.

### Что такое ИИ-провайдер и зачем он нужен

Для того чтобы бот «понимал» вопросы и давал умные ответы, нужны две ИИ-модели:

- **LLM (языковая модель)** — «мозг» бота, который читает вопрос и формулирует ответ. Похоже на ChatGPT, но может работать локально.
- **Embedding-модель** — модель, которая «понимает» смысл текста и помогает найти нужный документ среди загруженных. Работает «в фоне», пользователь её не видит.

Доступны три варианта, где запускать эти модели:

| Провайдер | Как работает | Стоимость | Требования |
|---|---|---|---|
| **Ollama** (рекомендуется) | Модель работает прямо на вашем компьютере | Бесплатно | ОЗУ от 8 ГБ, при первом запуске скачивает ~4–6 ГБ |
| **OpenAI** | Модель работает в облаке OpenAI (как ChatGPT) | Платно (по API-ключу) | Нужен интернет и ключ API от openai.com |
| **llama.cpp** | Модель работает локально, максимальный контроль над моделью | Бесплатно | Нужна ручная установка llama.cpp, файлы моделей в формате `.gguf` |

**Если вы запускаете впервые — выбирайте Ollama** (вариант по умолчанию).

## 2. Установить Docker

Docker — программа, которая запускает вспомогательные сервисы (базу данных и файловое хранилище) в изолированных контейнерах. Вам не нужно разбираться в том, как это работает — Docker всё сделает сам.

### macOS

1. Откройте браузер, перейдите на https://www.docker.com/products/docker-desktop/
2. Нажмите **«Download for Mac»**.
   - Если у вас Mac на M1/M2/M3/M4 — выберите **Apple Silicon**.
   - Если у вас старый Mac на Intel — выберите **Intel**.
3. Откройте скачанный `.dmg` файл.
4. Перетащите иконку Docker в папку **Программы** (Applications).
5. Запустите **Docker** из папки Программы. При первом запуске macOS может попросить подтвердить открытие — нажмите **«Открыть»**.
6. Дождитесь, пока иконка Docker в верхней панели (рядом с часами) перестанет мигать — это значит Docker готов к работе.

### Linux (Debian/Ubuntu)

Откройте **Терминал** и выполните команды по одной:

```bash
# Обновить список пакетов
sudo apt-get update

# Установить зависимости
sudo apt-get install -y ca-certificates curl gnupg

# Добавить ключ Docker
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# Добавить репозиторий Docker
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Установить Docker
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Разрешить запуск Docker без sudo
sudo usermod -aG docker $USER
```

**После последней команды выйдите из системы и войдите обратно** (или перезагрузите компьютер).

Проверьте, что Docker работает:

```bash
docker --version
```

Должна появиться строка вида `Docker version 27.x.x ...`

## 3. Установить uv

`uv` — менеджер Python-пакетов. Он автоматически скачает нужную версию Python и все зависимости проекта. Нужен только для вариантов запуска через bash-скрипты (варианты А и В).

### macOS и Linux

Откройте **Терминал** и выполните:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

После установки **закройте и откройте Терминал заново**, чтобы команда `uv` стала доступна.

Проверьте установку:

```bash
uv --version
```

Должна появиться строка вида `uv 0.x.x`.

> Для вариантов Б и Г (запуск через Docker) `uv` не нужен.

## 4. Установить Ollama (если используете локальный ИИ)

Ollama — программа, которая скачивает и запускает ИИ-модели прямо на вашем компьютере. Нужна только если вы выбрали **Ollama** как провайдер (вариант по умолчанию).

Если вы планируете использовать **OpenAI** — этот шаг пропустите.

### macOS

1. Перейдите на https://ollama.com/download
2. Нажмите **«Download for macOS»**.
3. Откройте скачанный файл и установите Ollama.
4. Запустите **Ollama** из папки Программы.

### Linux (Debian/Ubuntu)

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

Проверьте установку:

```bash
ollama --version
```

> **Важно:** При первом запуске скрипта Ollama автоматически скачает языковую модель (~2–3 ГБ) и embedding-модель (~2–3 ГБ). Это занимает время — зависит от скорости интернета. Дождитесь завершения.

## 5. Установить llama.cpp (если используете llamacpp)

llama.cpp — продвинутый вариант локального запуска ИИ-моделей. В отличие от Ollama, вы сами кладёте файлы моделей в папку `models/` и управляете серверами напрямую. Подходит, если вам нужен максимальный контроль над потреблением памяти и скоростью.

> Если вы используете **Ollama** или **OpenAI** — этот шаг пропустите.

### Установить llama-server

#### macOS (через Homebrew)

```bash
brew install llama.cpp
```

> Если Homebrew не установлен: перейдите на https://brew.sh и следуйте инструкции.

#### Linux (сборка из исходников)

```bash
sudo apt-get install -y build-essential cmake
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp
cmake -B build
cmake --build build --config Release -j$(nproc)
sudo cp build/bin/llama-server /usr/local/bin/
```

Проверьте установку:

```bash
llama-server --version
```

### Файлы моделей (.gguf)

llama.cpp использует модели в формате `.gguf`. Файлы нужно положить в папку `models/` в корне проекта.

По умолчанию скрипты ожидают два файла:

| Файл | Назначение | Размер |
|---|---|---|
| `models/Qwen3.5-4B-Q4_K_M.gguf` | LLM — языковая модель | ~2.5 ГБ |
| `models/Qwen3-Embedding-4B-Q4_K_M.gguf` | Embedding — поиск по документам | ~2.4 ГБ |

**Скрипты скачают модели автоматически** при первом запуске, если файлов нет. URL для скачивания можно переопределить через `LLM_MODEL_URL` / `EMBED_MODEL_URL` (см. раздел 12). Для ручного скачивания по умолчанию:

```bash
# LLM-модель
curl -L -o models/Qwen3.5-4B-Q4_K_M.gguf \
  https://huggingface.co/unsloth/Qwen3.5-4B-GGUF/resolve/main/Qwen3.5-4B-Q4_K_M.gguf

# Embedding-модель
curl -L -o models/Qwen3-Embedding-4B-Q4_K_M.gguf \
  https://huggingface.co/Qwen/Qwen3-Embedding-4B-GGUF/resolve/main/Qwen3-Embedding-4B-Q4_K_M.gguf
```

### Ускорение через GPU (определяется автоматически)

Скрипты сами определяют тип GPU и включают ускорение:

| Железо | Что используется |
|---|---|
| Mac на Apple Silicon (M1/M2/M3/M4) | Metal — все слои модели на GPU |
| NVIDIA (Linux) | CUDA — все слои модели на GPU |
| Всё остальное | CPU (без GPU-ускорения) |

Если нужно переопределить вручную — задайте переменные в `.env`:
```
LLM_N_GPU_LAYERS=99     # 99 = все слои на GPU, 0 = только CPU
EMBED_N_GPU_LAYERS=99
```

## 7. Настроить файл .env

Файл `.env` — это файл с настройками. Скрипты и Docker читают настройки именно из него.

```bash
cd ~/Projects/cafetera_hr_bot    # перейти в папку проекта
cp .env.example .env             # создать файл настроек из шаблона
```

Теперь откройте `.env` для редактирования:

### macOS

```bash
open -e .env
```

### Linux

```bash
nano .env
```

> В nano: редактируйте текст как обычно. Для сохранения нажмите `Ctrl+O`, затем `Enter`. Для выхода — `Ctrl+X`.

### Что нужно обязательно заполнить

| Параметр | Что вписать | Для чего |
|---|---|---|
| `ADMIN_API_KEY` | Любой длинный пароль, например `my-secret-key-2025` | Вход в админ-панель |
| `VK_ACCESS_TOKEN` | Токен вашего VK-сообщества | Нужен для VK-бота |
| `VK_GROUP_ID` | ID вашего VK-сообщества (число) | Нужен для VK-бота |

> **VK-токен и ID группы** нужны только если вы запускаете VK-бота. Для работы только с админ-панелью их можно не заполнять.

### Как получить VK_ACCESS_TOKEN и VK_GROUP_ID

1. Зайдите в своё сообщество ВКонтакте.
2. Перейдите в **Управление → Настройки → Работа с API**.
3. Нажмите **«Создать ключ»** и скопируйте токен.
4. `VK_GROUP_ID` — это числовой ID вашей группы (можно увидеть в адресной строке или в настройках группы).

### Настройка ИИ-провайдера в .env (необязательно)

Если вы хотите, чтобы скрипт не спрашивал каждый раз — укажите провайдера прямо в `.env`:

**Для Ollama (по умолчанию, бесплатно):**
```
LLM_PROVIDER=ollama
LLM_MODEL=qwen3.5:4b-q4_K_M
EMBEDDING_PROVIDER=ollama
EMBEDDING_MODEL=qwen3-embedding:4b-q4_K_M
```

**Для OpenAI (платно, нужен API-ключ):**
```
LLM_PROVIDER=openai
LLM_MODEL=gpt-4o-mini
LLM_API_KEY=sk-...ваш-ключ...
EMBEDDING_PROVIDER=openai
EMBEDDING_MODEL=text-embedding-3-small
EMBEDDING_API_KEY=sk-...ваш-ключ...
```

**Для llama.cpp (бесплатно, нужны файлы моделей в папке `models/`):**
```
LLM_PROVIDER=llamacpp
LLM_MODEL=local-model
EMBEDDING_PROVIDER=llamacpp
EMBEDDING_MODEL=qwen3-embedding
```

> При выборе llamacpp скрипты (`run_admin.sh`, `run_all.sh`) автоматически запускают два сервера: LLM на порту `8080` и Embedding на порту `8090`. Если файлов моделей нет — скрипт скачивает их с HuggingFace автоматически.

Остальные параметры (`DATABASE_URL`, `QDRANT_URL`, `S3_ENDPOINT_URL`) оставьте как есть — скрипты и Docker подставят правильные значения автоматически.

## 8. Выбрать способ запуска

Доступны четыре варианта. Прочитайте описание и выберите подходящий.

| Вариант | Что запускает | Как запускается | Кому подходит |
|---|---|---|---|
| **А** | Только админ-панель | bash-скрипт | Разработка, первый запуск |
| **Б** | Только админ-панель | Docker | Чистый Docker без uv |
| **В** | Админ-панель + VK-бот | bash-скрипт | Разработка с ботом |
| **Г** | Всё вместе | Docker Compose | Сервер, продакшн |

### Вариант А — только Админ-панель (bash-скрипт)

Самый простой способ для первого запуска на своём компьютере.

**Требования:** Docker, uv, Ollama (или OpenAI API-ключ, или llama.cpp + файлы моделей).

```bash
cd ~/Projects/cafetera_hr_bot
bash scripts/run_admin.sh
```

**Что произойдёт:**

1. Скрипт проверит, что Docker и uv установлены.
2. **Спросит, какой LLM-провайдер использовать:**
   ```
   Select LLM provider:
     1) ollama (default)
     2) openai
     3) llamacpp
   Enter choice [1-3, Enter=1]:
   ```
   Просто нажмите **Enter** для выбора Ollama. Для OpenAI введите `2` и вставьте API-ключ. Для llama.cpp введите `3` — скрипт сам запустит серверы.

3. **Спросит, какой Embedding-провайдер использовать** — снова нажмите **Enter**.

4. Установит Python-зависимости (1–2 минуты при первом запуске).

5. Запустит базы данных и хранилище (Qdrant, MinIO, PostgreSQL) через Docker.

6. Запустит Ollama и скачает модели (при первом запуске — несколько минут/часов в зависимости от интернета). Для llamacpp — запустит `llama-server` для LLM (порт 8080) и Embedding (порт 8090).

7. Запустит веб-сервер админ-панели.

**Готово, когда увидите:**
```
[admin] ==========================================
[admin] Starting admin server
[admin] ==========================================
[admin] Admin UI:    http://127.0.0.1:8000/documents
```

Для остановки нажмите **Ctrl+C** в том же окне Терминала.

Запускает админ-панель целиком через Docker. `uv` не нужен.

**Требования:** Docker, Ollama (или OpenAI API-ключ, или llama.cpp + файлы моделей). `uv` не нужен.


Скрипт задаст те же вопросы о провайдере LLM и Embedding, потом соберёт Docker-образ и запустит все сервисы.

**Готово, когда увидите:**
```
[admin-docker] Admin UI:    http://127.0.0.1:8000/documents
```

Для остановки нажмите **Ctrl+C**.

### Вариант В — Админ-панель + VK-бот вместе (bash-скрипт)

Запускает одновременно и веб-панель, и VK-бот в режиме polling (постоянное соединение с VK).

**Требования:** Docker, uv, Ollama (или OpenAI API-ключ, или llama.cpp + файлы моделей), заполненные `VK_ACCESS_TOKEN` и `VK_GROUP_ID` в `.env`.

```bash
cd ~/Projects/cafetera_hr_bot
bash scripts/run_all.sh
```

Скрипт задаст вопросы о провайдерах, запустит всю инфраструктуру, и затем покажет объединённые логи всех сервисов.

**Готово, когда увидите:**
```
[run-all] ==========================================
[run-all] All services are running!
[run-all] ==========================================
[run-all] Admin UI:  http://127.0.0.1:8000/documents
[run-all] VK Bot:    Running in polling mode
```

Для остановки нажмите **Ctrl+C** — скрипт автоматически остановит все сервисы.

---

### Вариант Г — Всё через Docker Compose (для сервера/продакшна)

Самый «чистый» способ: всё запускается в Docker-контейнерах. Подходит для постоянной работы на сервере. `uv` не нужен.

**Требования:** Docker, Ollama запущена на хосте (или OpenAI API-ключ прописан в `.env`).

> Перед запуском убедитесь, что в `.env` заполнены `ADMIN_API_KEY`, `VK_ACCESS_TOKEN`, `VK_GROUP_ID`, и выбраны провайдеры LLM/Embedding (см. [раздел 6](#настройка-ии-провайдера-в-env-необязательно)).

**Запуск всего стека:**

```bash
cd ~/Projects/cafetera_hr_bot
docker compose up -d --build
```

Это запустит:
- **PostgreSQL** — база данных
- **Qdrant** — база знаний для поиска по документам
- **MinIO** — файловое хранилище
- **Админ-панель** — веб-интерфейс (порт 8000)
- **VK-бот** — бот в режиме polling

Посмотреть что происходит (логи):

```bash
docker compose logs -f              # логи всех сервисов
docker compose logs -f admin        # только админка
docker compose logs -f vk_bot       # только VK-бот
```

Остановить все сервисы:

```bash
docker compose down
```

**Запустить только нужные части:**

```bash
# Только инфраструктура + админка (без VK-бота)
docker compose up -d --build qdrant minio postgres admin

# Только инфраструктура + VK-бот (без админки)
docker compose up -d --build qdrant minio postgres vk_bot
```

> **Примечание об Ollama в Docker:** Ollama должна быть запущена на вашем компьютере (хосте). Docker-контейнеры обращаются к ней по адресу `host.docker.internal:11434` автоматически — ничего дополнительно настраивать не нужно.

## 10. Остановить сервисы

| Способ запуска | Как остановить |
|---|---|

## 11. Частые проблемы

### «docker: command not found»
Docker не установлен или не запущен. Вернитесь к [шагу 2](#2-установить-docker).

### «uv: command not found»
Закройте Терминал и откройте заново. Если не помогло — повторите [шаг 3](#3-установить-uv).

### «ADMIN_API_KEY is not set»
Откройте файл `.env` и убедитесь, что строка `ADMIN_API_KEY=` содержит значение (не пустая).

### «Qdrant failed to start» / «MinIO failed to start» / «PostgreSQL failed to start»
- Убедитесь, что Docker запущен.
- Проверьте, не заняты ли порты другими программами:
  ```bash
  lsof -i :6333   # Qdrant
  lsof -i :9000   # MinIO
  lsof -i :5432   # PostgreSQL
  ```
- Посмотрите логи Docker:
  ```bash
  docker compose logs
  ```

### «Ollama is not installed»
Вернитесь к [шагу 4](#4-установить-ollama-если-используете-локальный-ии) и установите Ollama.

### Модель скачивается очень долго
При первом запуске Ollama скачивает модели — суммарно около 4–6 ГБ. Это нормально. Дождитесь завершения — при следующем запуске модели уже будут скачаны.

### Порт 8000 уже занят
Запустите на другом порту (например, 8080):
```bash
ADMIN_PORT=8080 bash scripts/run_admin.sh
```
Тогда AdminUI будет доступна по адресу http://127.0.0.1:8080/documents

### VK-бот не отвечает в группе
1. Проверьте, что `VK_ACCESS_TOKEN` и `VK_GROUP_ID` правильно заполнены в `.env`.
2. Убедитесь, что бот запущен: `docker compose logs -f vk_bot`
3. Проверьте, что в настройках VK-сообщества включены «Сообщения».

### Ошибка при запуске через Docker на Linux: «host.docker.internal»
На Linux это имя хоста не всегда работает автоматически. Скрипт и `docker-compose.yml` добавляют `extra_hosts: host.docker.internal:host-gateway` — это должно решить проблему. Если нет — проверьте версию Docker: `docker --version` (нужна 20.10+).

### Ошибка: «llamacpp: llama-server not found» или порт 8080/8090 уже занят
Что проверить:
1. `llama-server` установлен: `llama-server --version`
2. Порты свободны:
   ```bash
   lsof -i :8080   # LLM-сервер
   lsof -i :8090   # Embedding-сервер
   ```
3. Файлы моделей есть в `models/`: `ls -lh models/`
4. Если сервер упал — посмотрите логи: `/tmp/llama_llm.log` и `/tmp/llama_embed.log`

## 12. Запуск серверов llama.cpp вручную

Скрипты `run_admin.sh` и `run_all.sh` запускают `llama-server` автоматически. Но если вам нужно запустить серверы отдельно (например, в отдельном окне терминала):

**LLM-сервер** (порт 8080):
```bash
bash scripts/run_llama_llm.sh
```

**Embedding-сервер** (порт 8090):
```bash
bash scripts/run_llama_embeddings.sh
```

Оба скрипта автоматически:
- Определяют тип GPU (Metal на Mac M1/M2/M3/M4, CUDA на NVIDIA, иначе CPU) и включают ускорение
- Скачивают файл модели с HuggingFace, если его нет в `models/`

Параметры можно переопределить через переменные среды:

```bash
# Использовать другой файл LLM-модели:
LLM_MODEL_PATH=./models/my-llm.gguf bash scripts/run_llama_llm.sh

# Скачать LLM-модель с другого URL вместо HuggingFace по умолчанию:
LLM_MODEL_URL=https://example.com/my-model.gguf bash scripts/run_llama_llm.sh

# Использовать другой файл Embedding-модели:
EMBED_MODEL_PATH=./models/my-embed.gguf bash scripts/run_llama_embeddings.sh

# Скачать Embedding-модель с другого URL:
EMBED_MODEL_URL=https://example.com/my-embed.gguf bash scripts/run_llama_embeddings.sh

# Принудительно только CPU (без GPU):
LLM_N_GPU_LAYERS=0 bash scripts/run_llama_llm.sh
EMBED_N_GPU_LAYERS=0 bash scripts/run_llama_embeddings.sh
```
