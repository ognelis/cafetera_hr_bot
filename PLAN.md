# Cafetera HR-бот — план разработки

***

## Фаза 1 — Кликовый навигатор + Long Polling + RAG-заглушки

**Цель:** рабочий бот, которым можно пользоваться без RAG.
Все сценарии доступны. RAG-разделы возвращают заглушку и кнопку «Написать в HR».

***

### Блок 1 — Фундамент

**1.1 Скаффолдинг проекта**
- vkbottle 4.x, long polling через `scripts/polling_vk.py`
- `pydantic-settings`, `.env.example`
- Структура: `app/integrations/vk/`, `app/domain/`, `scripts/`
- Инициализация ресурсов перед запуском polling

**1.2 Состояния и контекст**
- Состояния экранов → `bot.state_dispenser` (in-memory, встроен в vkbottle)
- Enum `MyStates` для каждого многошагового сценария
- Контекст диалога S-70 → `CtxStorage` (in-memory, встроен в vkbottle)

**1.3 Роутер сообщений**
- Навигация → payload-роутинг через декораторы vkbottle
- Состояние-зависимые хендлеры через `state=MyStates.X`
- Fallback на текстовый ввод → заглушка или главное меню

**1.4 Навигационные хелперы**
- `send_main_menu(user_id)` — S-01
- Кнопки «⬅ Назад», «🏠 Главное меню», «💬 Написать в HR» доступны на каждом экране (UXR-5)
- Единая функция сборки клавиатуры с сервисными кнопками

***

### Блок 2 — P0: кликовые сценарии (без RAG)

**2.1 🏠 Главное меню (S-01)**
- FR-1: 7 разделов меню согласно разделу 12 ТЗ
- Приветственное сообщение при `/start`:
  «Привет! Я HR-бот Cafetera. Помогу найти документ, инструкцию или ответ на кадровый вопрос.»

**2.2 👤 Приём сотрудника (S-10, S-11)**
- FR-2, FR-3, FR-4, FR-14
- Шаг 1: выбор юрлица (4 кнопки с полными наименованиями, NFR-7)
- Шаг 2: выбор действия:
  - ✅ Чек-лист документов — статичный текст по юрлицу
  - 📄 Шаблон трудового договора — дисклеймер (FR-17) → файл Word
  - 🗒️ Онбординг-чек-лист — статичный текст

**2.3 🚪 Увольнение — кликовая часть (S-20)**
- FR-6: ✅ Чек-лист последнего рабочего дня — статичный текст
- S-21b: 📥 Обходной лист — дисклеймер + скачать Word-файл (FR-17)
- Кнопки перехода к RAG-сценариям — заглушки из Блока 3

**2.4 🏖 Отпуск — шаблон заявления (S-30 partial)**
- FR-8: выбор юрлица → дисклеймер → 📄 файл Word
- Кнопки перехода к RAG-сценариям — заглушки из Блока 3

**2.5 💬 Формирование обращения в HR (S-70)**
- FR-16: диалог из 5–6 шагов
- Поля: ФИО сотрудника, тема, суть, юрлицо, срочность
- Сборка структурированного текста обращения
- Подтверждение пользователем → показ готового текста + «📋 Скопировать»
- FR-18: кнопка «💬 Написать в HR» доступна из любого экрана

### Блок 3 — P0: RAG-заглушки

**3.1 Сервис RAG-заглушки**
- Единая функция `rag_stub(topic)` — возвращает текст-заглушку
- Текст: «ℹ️ [тема] — ответ формируется через базу знаний (в разработке). Обратитесь в HR или попробуйте позже.»
- Кнопки: [💬 Написать в HR] [🏠 Главное меню]

**3.2 🚪 Увольнение по собственному желанию** — FR-5 (RAG stub)
**3.3 🏖 Порядок оформления отпуска** — FR-7 (RAG stub)
**3.4 🏆 Условия премирования** — FR-10 (RAG stub)

***

### Блок 4 — P1: кликовые сценарии и состояния ошибок

**4.1 💰 Оплата и премии (S-40)**
- Меню раздела + переходы к RAG-заглушкам (FR-9, FR-10)

**4.2 🏥 Больничный / ЭЛН (S-50)** — FR-13 (RAG stub)
**4.3 📝 Испытательный срок (S-60)** — FR-15 (RAG stub)

**4.4 ❓ Задать вопрос**
- Приём свободного текста → RAG stub → кнопки навигации

**4.5 Состояния ошибок**
- S-ERR-01: документ временно недоступен → FR-19 текст + [💬 HR]
- S-ERR-02: нет точного ответа в базе знаний → текст + [💬 HR]
- S-ERR-03: данные требуют интеграции → NFR-1 текст + [💬 HR]

**4.6 Out-of-scope заглушки** — FR-20
- Запрос по переводу, дисциплине → RAG stub + [💬 Написать в HR]
- Запрос персональных данных → S-ERR-03

## Фаза 2 — Базовый RAG

**Цель:** заменить заглушки реальными ответами из Word-документов.
Кликовая часть не трогается.

***

### Блок 6 — RAG-инфраструктура

**6.0 FastAPI + webhook**
- Добавить FastAPI — нужен для webhook VK и эндпоинта загрузки документов
- Перенести инициализацию ресурсов в lifespan
- Webhook-эндпоинт для VK вместо long polling (прод-деплой)

**6.1 Qdrant в docker-compose**
- Добавить сервис Qdrant в существующий `docker-compose.yml`
- Создать коллекцию `hr_documents`

**6.2 Пайплайн ингестиции Word-документов**
- `scripts/ingest.py`: python-docx → чанки → эмбеддинги → Qdrant
- Только .docx (NFR-3: PDF-сканы не используются)
- Метаданные: имя файла, раздел, юрлицо (если применимо)

**6.3 LangChain ретривер**
- `app/rag/retriever.py`: dense retrieval из Qdrant
- `app/rag/prompts.py`: системный промпт — краткий, пошаговый, без персональных данных
- `app/rag/chain.py`: retriever + prompt + LLM
- Инициализация в lifespan

### Блок 9 — Свободные вопросы и edge cases

**9.1 ❓ «Задать вопрос» → реальный RAG**
- Свободный текст → RAG chain → ответ
- Если ответ связан с кликовым сценарием → предложить перейти в него
- Если RAG не нашёл ответ → S-ERR-02 + [💬 Написать в HR]

**9.2 Фоновые темы через RAG** (без кнопочного сценария, раздел 10 ТЗ)
- Перевод сотрудника → RAG + «По этой теме рекомендуем согласовать оформление напрямую с HR.»
- Дисциплина → RAG + [💬 Написать в HR]
- 🚪 Увольнение за прогул → RAG + [💬 Написать в HR]

|---|---|---|
